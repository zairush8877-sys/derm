# Подключение SMS для кодов входа — 15 минут

Сейчас вход по коду работает в демо-режиме: код показывается прямо на экране
(бесплатно, удобно для разработки). Один ключ переключает на настоящие SMS.

## Вариант 1 (рекомендуется): SMS.ru

1. Зарегистрируйтесь на **sms.ru** (по номеру телефона).
2. В личном кабинете: **Настройки → API** — скопируйте `api_id`
   (вид: `XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX`).
3. Пополните баланс (от 100 ₽ хватит на старт).
4. Впишите в `.env`:
   ```
   SMS_API_KEY=ваш-api_id
   ```
5. Перезапустите приложение — демо-режим выключится сам.

**Имя отправителя.** По умолчанию SMS приходят с общего номера/имени провайдера.
Чтобы приходило «AURA» — в кабинете SMS.ru подайте заявку на имя отправителя
(«Отправители»), после модерации добавьте в `.env`: `SMS_SENDER=AURA`.

**Цена:** ~4–7 ₽ за SMS по РФ (зависит от оператора и наличия имени отправителя).

## Вариант 2: SMSC.ru

1. Регистрация на **smsc.ru**, в настройках создайте **API-пароль**
   (не используйте основной пароль аккаунта).
2. В `.env`:
   ```
   SMS_PROVIDER=smsc
   SMS_LOGIN=ваш-логин
   SMS_PASSWORD=api-пароль
   ```

## Как проверить

1. Откройте `/auth`, введите **свой** номер, нажмите «Получить код».
2. Плашки «Демо-режим» быть не должно — код придёт настоящей SMS.
3. В логах приложения будет строка «Отправка SMS через smsru…».

## Встроенная защита (уже работает, настраивать не нужно)

| Защита | Лимит |
|---|---|
| Повторная отправка на номер | не чаще 1 раза в 60 секунд |
| SMS на один номер | максимум 5 в час |
| Попыток ввода кода | 5, дальше нужен новый код |
| Срок жизни кода | 5 минут |

Это защита от «SMS-бомбинга» (когда злоумышленник вводит чужие номера и
сжигает ваш баланс) и от перебора кода.

## Частые ошибки

| Симптом | Причина |
|---|---|
| «Не удалось отправить SMS» | Нет денег на балансе провайдера / неверный api_id |
| Код приходит долго | Задержка оператора; проверьте статус в кабинете провайдера |
| Осталась плашка «Демо-режим» | Не перезапустили приложение после правки `.env` |
