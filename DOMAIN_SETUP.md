# Постоянный домен + HTTPS для Aura — пошагово

Сейчас сайт живёт по временной ссылке `*.trycloudflare.com`. Домен даёт
**постоянный красивый адрес** (например, `aura-wellness.ru`) с HTTPS-замочком,
который открывается у всех и не режется блокировками. ~20 минут.

## Шаг 1. Купить домен

1. В панели Timeweb слева: **«Домены и SSL»** → «Зарегистрировать домен»
   *(можно и на reg.ru — без разницы)*.
2. Придумайте имя в зоне **.ru** (от ~200 ₽/год) — например `aura-wellness.ru`.
   Проверьте, что свободно, и оплатите.

## Шаг 2. Направить домен на сервер (A-запись)

1. Откройте настройки купленного домена → раздел **«DNS»** (или «Управление зоной»).
2. Создайте запись типа **A**:
   - **Имя/Хост:** `@` (это корень домена)
   - **Значение/IP:** `72.56.13.39`
   - TTL — по умолчанию
3. Добавьте ещё одну A-запись для `www`:
   - **Имя:** `www` → **IP:** `72.56.13.39`
4. Сохраните. DNS обновляется от нескольких минут до пары часов
   (обычно 15–30 мин). Проверить можно на **dnschecker.org** — введите домен,
   должен показать `72.56.13.39`.

## Шаг 3. Перезапустить сайт с доменом

Когда A-запись заработала — в **веб-консоли Timeweb** (root) выполните
（подставьте свой домен）:

```
cd /root/aura && DOMAIN='ваш-домен.ru' bash scripts/vps-bootstrap.sh
```

Caddy сам получит бесплатный SSL-сертификат Let's Encrypt (замочек) за 1–2 минуты.

> Если домен с `www` — Caddy можно настроить на оба; напишите мне, добавлю
> в `Caddyfile` строку с редиректом `www → без www`.

## Шаг 4. Проверить

Откройте **https://ваш-домен.ru** — сайт откроется с зелёным замочком.
Теперь можно:
- отключить временный туннель: `docker rm -f aura-tunnel`
- давать всем этот постоянный адрес.

## Частые вопросы

| Вопрос | Ответ |
|---|---|
| Домен не открывается сразу | DNS ещё обновляется — подождите 30–60 мин, проверьте на dnschecker.org |
| «Небезопасно» / нет замочка | Сертификат ещё выпускается (1–2 мин) или домен не долетел до сервера |
| Хочу почту на домене (info@…) | Настраивается отдельно в Timeweb → «Почта», скажите — помогу |

## Почему домен решает проблему блокировки

Временный IP-адрес по `http://` российские сети часто сбрасывают. Домен с
**HTTPS** шифрует трафик — провайдер не видит содержимое и не режет соединение.
Плюс это профессионально и вызывает доверие у покупателей.
