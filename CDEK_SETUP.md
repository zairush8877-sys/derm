# Доставка СДЭК и уведомления о заказах

## Как работает сейчас (до ИП) — «ручной» режим

1. Покупатель оформляет заказ: адрес, телефон, комментарий, способ доставки
   (фиксированные тарифы: курьер 390 ₽, ПВЗ 250 ₽, почта 350 ₽).
2. Вам приходит **уведомление в Telegram** с составом заказа и адресом
   (после настройки, см. ниже).
3. Вы оформляете отправку в приложении СДЭК как частное лицо.
4. В админке жмёте **«📦 трек»** у заказа и вписываете трек-номер —
   покупатель автоматически получает ссылку отслеживания cdek.ru.
5. Двигаете статус: собирается → отправлен → доставлен (уведомления сами).

## Уведомления о новых заказах в Telegram (5 минут)

1. В Telegram найдите **@BotFather** → команда `/newbot` → придумайте имя →
   скопируйте **токен** (вида `1234567:AA...`).
2. Напишите своему новому боту любое сообщение (это «открывает» диалог).
3. Найдите **@userinfobot** → он покажет ваш **chat id** (число).
4. Впишите в `.env` на сервере (или пришлите мне — вставлю):
   ```
   TELEGRAM_BOT_TOKEN=токен-из-botfather
   TELEGRAM_CHAT_ID=ваш-chat-id
   ```
5. После перезапуска каждый новый заказ будет прилетать вам в Telegram.

## Автоматический СДЭК (после оформления ИП)

1. Заключите договор на **cdek.ru** (раздел «Интернет-магазинам») — нужен ИП/ООО.
2. В личном кабинете СДЭК: **Интеграция → API** → получите Account и Secure password.
3. В `.env`:
   ```
   CDEK_ACCOUNT=...
   CDEK_PASSWORD=...
   CDEK_FROM_CITY=44   # откуда отправляете (44 Москва, 137 СПб)
   ```
4. После этого стоимость и срок доставки считаются **по реальным тарифам СДЭК**
   автоматически (модуль app/shop/cdek.py уже готов). Если API недоступен —
   сайт тихо откатывается на фиксированные тарифы, заказы не ломаются.

Дальнейшие шаги (по мере роста): выбор ПВЗ на карте, автоматическое создание
накладной и получение трек-номера из API — скажите, добавлю, когда будет договор.
